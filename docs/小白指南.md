# 实验室管理系统 - 小白使用指南

> 不会编程？不懂Docker？没关系！跟着本文一步步操作，10分钟就能把系统跑起来。

---

## 一、你需要准备什么

- 一台电脑（Windows 10/11 或 macOS）
- 至少 8GB 内存
- 能上网

---

## 二、选择你的启动方式

### 方式 A：裸机运行（推荐小白，无需 Docker）⭐

不需要安装 Docker，直接在你的 Windows 上运行。脚本会自动帮你安装 Java、Node.js、MySQL。

1. 进入项目 `scripts` 文件夹
2. 双击运行 `raw_start.bat`（会自动请求管理员权限）
3. 在弹出的蓝底菜单中，输入 `1` 回车 → 一键安装+部署
4. 等5-10分钟（首次需要下载依赖）
5. 浏览器打开 http://localhost:3000

> 这个脚本首次运行会自动通过 winget 安装 Java 21、Node.js 20、MySQL 8.0、Maven，全程无需手动下载。

### 方式 B：Docker 部署（需要先安装 Docker Desktop）

如果你已经安装了 Docker Desktop，可以使用这种方式。

**Windows 用户**：双击 `scripts\init-deploy.bat`，等待3-5分钟即可。

**macOS/Linux 用户**：
```bash
chmod +x scripts/init-deploy.sh
./scripts/init-deploy.sh
```

## 五、日常操作

### 查看系统是否在运行
```bash
docker ps
```
看到 `lab-mysql`、`lab-redis`、`lab-backend`、`lab-frontend` 都是 `Up` 状态就是正常的。

### 重启系统
```bash
docker restart lab-backend lab-frontend
```

### 查看后端日志
```bash
docker logs -f lab-backend
```
按 `Ctrl+C` 退出日志查看。

### 完全停止系统
```bash
docker stop lab-frontend lab-backend lab-redis lab-mysql
```

### 再次启动
```bash
docker start lab-mysql lab-redis lab-backend lab-frontend
```
> 注意启动顺序：先MySQL → 再Redis → 再后端 → 最后前端

### 修改管理员密码
1. 用管理员账号登录系统
2. 点击右上角头像 → 个人中心
3. 在"修改密码"区域输入新密码

## 六、遇到问题的解决办法

### 问题1：Docker Desktop 打不开
- 检查是否开启了虚拟化（VT-x/AMD-V），进入BIOS打开
- Windows：控制面板 → 程序和功能 → 启用或关闭Windows功能 → 勾选 Hyper-V

### 问题2：启动时提示端口被占用
```bash
# 查看占用 8081 端口的程序
netstat -ano | findstr :8081   (Windows)
lsof -i :8081                   (Mac/Linux)
```
关闭占用端口的程序，或者修改 `.env` 文件里的 `BACKEND_PORT` 改成其他端口。

### 问题3：http://localhost 打不开
1. 确认 Docker Desktop 在运行
2. 运行 `docker ps` 确认 `lab-frontend` 是 Up 状态
3. 如果前端没启动，运行 `docker start lab-frontend`

### 问题4：登录提示"网络错误"
1. 检查后端是否启动：`docker ps | grep lab-backend`
2. 查看后端日志：`docker logs lab-backend --tail 30`
3. 如果看到 `Connection refused`，说明MySQL还没就绪，等30秒再试

### 问题5：数据库被清空了
```bash
# 重新初始化数据库
docker exec -i lab-mysql mysql -u root -p你的密码 < backend/src/main/resources/db/schema.sql
```

## 七、进阶：使用 Jenkins 自动部署

如果你有 Jenkins 服务器，可以使用内置的 CI/CD 流水线：

1. 在 Jenkins 中新建 Pipeline 项目
2. Pipeline definition 选择 "Pipeline script from SCM"
3. SCM 选择 Git，填入仓库地址
4. Script Path 填写 `Jenkinsfile`
5. 构建前在 Jenkins 凭据管理中配置：
   - `lab-mysql-root-password`
   - `lab-mysql-password`
   - `lab-jwt-secret`
6. 点击 Build with Parameters，填写 `DEPLOY_HOST` 为目标服务器IP
7. 点击构建，Jenkins 会自动完成整个部署流程

## 八、（可选）启用监控面板

如果想查看系统运行状态（CPU、内存、请求量等），可以在 `.env` 中设置 `MONITOR_ENABLED=true`，重新运行部署脚本即可自动部署。

启动后访问：
- Grafana 仪表盘：http://localhost:3001（账号 admin/admin123）
- Prometheus 指标：http://localhost:9090

不需要时关闭：
```bash
cd monitor
docker-compose -f docker-compose.monitoring.yml down
```
