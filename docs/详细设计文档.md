## 2 需求概述

### 2.1 功能需求

系统面向三类用户：**学生**、**教师**、**管理员**，包含以下核心模块：

1. **用户管理模块**：用户注册、登录、JWT认证、权限控制、账号维护
2. **实验室管理模块**：实验室信息增删改查、状态管理
3. **设备管理模块**：设备信息维护、借用归还流程
4. **预约管理模块**：在线预约、自动冲突检测、审批流程
5. **耗材管理模块**：库存管理、出入库、库存预警
6. **数据统计模块**：使用率统计、图表展示、Excel导出
7. **辅助功能**：公告管理、操作日志

### 2.2 非功能需求

- 性能：普通查询响应 ≤ 2秒，写操作 ≤ 1秒，支持 ≥ 50并发
- 安全性：密码BCrypt加密、SQL注入防护、XSS防护、JWT认证、权限控制、操作日志
- 可扩展性：模块松耦合、前后端分离、预留扩展字段
- 可用性：界面简洁、操作直观、错误提示清晰、二次确认
- 可靠性：保证数据一致性，预约冲突检测准确，支持数据备份

### 3.1 分层架构

```
┌─────────────────────────────────────────────────────────────┐
│                        浏览器(表示层                          │
│  ┌─────────────┐                                             │
│  │  Vue 前端   │ ←─  HTTP RESTful API                      │
│  └─────────────┘                                             │
└─────────────────────────────────────────────────────────────┘
                              ↑
                              │ HTTP (JSON)
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                     Spring Boot 应用服务器                    │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐  │
│  │ Controller  │─→│   Service   │─→│       Mapper        │  │
│  │  (请求处理) │  │ (业务逻辑)   │  │    (数据访问)       │  │
│  └─────────────┘  └─────────────┘  └─────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
                              ↑
                              │
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                        MySQL 数据库                           │
│                     (数据持久化                              │
└─────────────────────────────────────────────────────────────┘
```

**分层职责：

| 层级 | 职责 |
|------|------|
| Controller层 | 接收HTTP请求、参数校验、响应封装，不处理业务逻辑 |
| Service层 | 实现业务逻辑，如预约冲突检测、权限验证、数据校验 |
| Mapper层 | 数据库CRUD操作，使用MyBatis-Plus |

## 4 数据库设计

### 4.1 E-R图关系

```
用户表(user)
    ├─→ 预约记录表(reservation)  (一对多，一个用户多个预约)
    ├─→ 设备表(device)            (一对多，一个用户借用多个设备)
    ├─→ 耗材出入库记录(consumable_log) (一对多)
    ├─→ 公告表(announcement)    (一对多，一个管理员发布多个公告)
    └─→ 操作日志表(operation_log) (一对多)

实验室表(lab)
    ├─→ 预约记录表(reservation)  (一对多)
    └─→ 设备表(device)           (一对多，一个实验室多个设备)

耗材表(consumable)
    └─→ 耗材出入库记录(consumable_log) (一对多)
```

### 4.3 数据库SQL脚本

位置：`backend/src/main/resources/db/schema.sql`

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS lab_management DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

USE lab_management;

-- 1. 用户表
CREATE TABLE `user` (
    `id` BIGINT AUTO_INCREMENT PRIMARY KEY,
    `username` VARCHAR(50) NOT NULL UNIQUE COMMENT '学号/工号',
    `password` VARCHAR(255) NOT NULL COMMENT '加密密码',
    `real_name` VARCHAR(50) COMMENT '真实姓名',
    `role` ENUM('STUDENT', 'TEACHER', 'ADMIN') NOT NULL COMMENT '角色',
    `phone` VARCHAR(20) COMMENT '联系电话',
    `email` VARCHAR(100) COMMENT '邮箱',
    `status` ENUM('ENABLED', 'DISABLED') DEFAULT 'ENABLED' COMMENT '账号状态',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '注册时间',
    `last_login_time` DATETIME COMMENT '最后登录时间',
    `remark` VARCHAR(500) COMMENT '备注',
    `create_by` BIGINT COMMENT '创建人',
    `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    `deleted` TINYINT DEFAULT 0 COMMENT '逻辑删除',
    INDEX idx_username(username)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';

-- 2. 实验室表
CREATE TABLE `lab` (
    `id` BIGINT AUTO_INCREMENT PRIMARY KEY,
    `name` VARCHAR(100) NOT NULL UNIQUE COMMENT '实验室名称',
    `location` VARCHAR(100) COMMENT '位置',
    `capacity` INT COMMENT '容纳人数',
    `device_count` INT DEFAULT 0 COMMENT '设备数量',
    `status` ENUM('FREE', 'OCCUPIED', 'MAINTENANCE') DEFAULT 'FREE' COMMENT '当前状态',
    `director` VARCHAR(50) COMMENT '负责人',
    `phone` VARCHAR(20) COMMENT '联系电话',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    `deleted` TINYINT DEFAULT 0 COMMENT '逻辑删除',
    `remark` VARCHAR(500) COMMENT '备注',
    INDEX idx_status(status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='实验室表';

-- 3. 设备表
CREATE TABLE `device` (
    `id` BIGINT AUTO_INCREMENT PRIMARY KEY,
    `name` VARCHAR(100) NOT NULL COMMENT '设备名称',
    `model` VARCHAR(100) COMMENT '型号',
    `serial_number` VARCHAR(100) COMMENT '序列号',
    `lab_id` BIGINT NOT NULL COMMENT '所属实验室ID',
    `purchase_date` DATE COMMENT '购买日期',
    `status` ENUM('NORMAL', 'BORROWED', 'MAINTENANCE', 'SCRAPPED') DEFAULT 'NORMAL' COMMENT '状态',
    `borrower_id` BIGINT COMMENT '借用人ID',
    `borrow_time` DATETIME COMMENT '借用时间',
    `expect_return_time` DATE COMMENT '预计归还时间',
    `return_time` DATETIME COMMENT '实际归还时间',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    `deleted` TINYINT DEFAULT 0 COMMENT '逻辑删除',
    `remark` VARCHAR(500) COMMENT '备注',
    INDEX idx_lab_id(lab_id),
    INDEX idx_status(status),
    INDEX idx_borrower_id(borrower_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='设备表';

-- 4. 预约记录表
CREATE TABLE `reservation` (
    `id` BIGINT AUTO_INCREMENT PRIMARY KEY,
    `user_id` BIGINT NOT NULL COMMENT '预约人ID',
    `lab_id` BIGINT NOT NULL COMMENT '实验室ID',
    `reservation_date` DATE NOT NULL COMMENT '预约日期',
    `start_time` TIME NOT NULL COMMENT '开始时间',
    `end_time` TIME NOT NULL COMMENT '结束时间',
    `purpose` VARCHAR(500) COMMENT '预约事由',
    `participant_count` INT COMMENT '参与人数',
    `status` ENUM('PENDING', 'APPROVED', 'REJECTED', 'CANCELED') DEFAULT 'PENDING' COMMENT '审批状态',
    `approver_id` BIGINT COMMENT '审批人ID',
    `approve_time` DATETIME COMMENT '审批时间',
    `approve_comment` VARCHAR(500) COMMENT '审批意见',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    `deleted` TINYINT DEFAULT 0 COMMENT '逻辑删除',
    INDEX idx_user_id(user_id),
    INDEX idx_lab_id(lab_id),
    INDEX idx_reservation_date(reservation_date),
    INDEX idx_status(status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='预约记录表';

-- 5. 耗材表
CREATE TABLE `consumable` (
    `id` BIGINT AUTO_INCREMENT PRIMARY KEY,
    `name` VARCHAR(100) NOT NULL COMMENT '耗材名称',
    `specification` VARCHAR(100) COMMENT '规格',
    `unit` VARCHAR(20) COMMENT '单位',
    `current_stock` DECIMAL(10,2) DEFAULT 0 COMMENT '当前库存',
    `warning_threshold` DECIMAL(10,2) DEFAULT 0 COMMENT '预警阈值',
    `location` VARCHAR(100) COMMENT '存放位置',
    `director` VARCHAR(50) COMMENT '负责人',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    `deleted` TINYINT DEFAULT 0 COMMENT '逻辑删除',
    `remark` VARCHAR(500) COMMENT '备注'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='耗材表';

-- 6. 耗材出入库记录表
CREATE TABLE `consumable_log` (
    `id` BIGINT AUTO_INCREMENT PRIMARY KEY,
    `consumable_id` BIGINT NOT NULL COMMENT '耗材ID',
    `operation_type` ENUM('IN', 'OUT') NOT NULL COMMENT '操作类型：入库/出库',
    `quantity` DECIMAL(10,2) NOT NULL COMMENT '数量',
    `operator_id` BIGINT NOT NULL COMMENT '操作人ID',
    `operation_time` DATETIME NOT NULL COMMENT '操作时间',
    `receiver` VARCHAR(50) COMMENT '领用人',
    `purpose` VARCHAR(500) COMMENT '用途',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `deleted` TINYINT DEFAULT 0 COMMENT '逻辑删除',
    INDEX idx_consumable_id(consumable_id),
    INDEX idx_operation_time(operation_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='耗材出入库记录表';

CREATE TABLE `announcement` (
    `id` BIGINT AUTO_INCREMENT PRIMARY KEY,
    `title` VARCHAR(200) NOT NULL COMMENT '标题',

    `publisher_id` BIGINT NOT NULL COMMENT '发布人ID',
    `publish_time` DATETIME COMMENT '发布时间',
    `is_top` TINYINT DEFAULT 0 COMMENT '是否置顶：0-否，1-是',
    `status` ENUM('DRAFT', 'PUBLISHED', 'CLOSED') DEFAULT 'PUBLISHED' COMMENT '状态',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
```

## 8 核心业务流程设计

### 8.1 登录流程

```
用户输入用户名密码
      ↓
前端发送POST /api/auth/login
      ↓
后端验证用户名密码
      ↓
验证通过 → 生成JWT Token（包含用户ID、角色）→ 返回给前端
      ↓
前端保存Token到localStorage → 跳转首页（根据角色）
      ↓
后续请求在Header中携带: Authorization: Bearer {token}
      ↓
后端JWT过滤器验证Token有效性 → 设置SecurityContext → 放行

密码错误 → 返回401
```

### 8.2 预约申请与冲突检测流程

```
用户选择 实验室 + 日期
      ↓
前端请求 GET /api/reservation/busy?labId=x&date=y
      ↓
后端查询该实验室该日期状态为APPROVED的所有预约，返回已占用时间段列表
      ↓
前端日历标记不可选时间段，用户选择空闲时间段提交申请
      ↓
后端开启事务
      ↓
查询是否存在冲突：
SELECT COUNT(*) FROM reservation
WHERE lab_id = ? AND reservation_date = ?
AND status = 'APPROVED'
AND (start_time < 新结束时间 AND end_time > 新开始时间)
      ↓
COUNT > 0 → 冲突 → 返回错误提示
COUNT = 0 → 无冲突 → 插入预约记录，状态PENDING → 返回成功
      ↓
事务提交
```

> 说明：使用事务保证并发情况下不会出现数据不一致
```

### 8.3 预约审批流程

```
待审批预约列表展示
      ↓
审批人点击 通过/拒绝
      ↓
后端调用审批接口，检查原状态必须是PENDING
      ↓
修改状态为APPROVED/REJECTED，记录审批人、审批时间、意见
      ↓
返回成功，前端刷新列表
```

### 8.4 设备借用流程

```
用户查看设备列表，状态NORMAL可借用
      ↓
用户申请借用某设备
      ↓
后端检查状态是否为NORMAL
      ↓
不是NORMAL → 报错不可借用
      ↓
是NORMAL → 更新状态为BORROWED，记录借用人、借用时间 → 返回成功
```

### 8.5 设备归还流程

```
用户归还设备
      ↓
后端更新状态为NORMAL，清空借用人，记录归还时间 → 返回成功

### 8.6 耗材领用流程

```
用户选择耗材，输入领用数量
      ↓
后端查询当前库存
      ↓
领用数量 > 当前库存 → 报错库存不足
      ↓
领用数量 ≤ 当前库存 → 扣减库存
      ↓
插入出库记录 → 操作类型OUT
      ↓
检查扣减后库存是否 ≤ 预警阈值 → 是 → 记录预警日志
      ↓
返回结果
```
